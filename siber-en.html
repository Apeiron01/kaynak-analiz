Delivery model

One project, client-based cloning, separate API and data layers

The core codebase stays the same for every client. Prompting, channels, integrations, knowledge base and access permissions are configured separately per client so delivery is faster and maintenance stays centralized.

How does it work?

We clone the base project, define the client's API keys, connect the required channels, load the corporate document set and adjust the bot behavior to the brand tone.

What changes per client?

The prompt, integration map, CRM fields, report format, department permissions, escalation rules and knowledge base are isolated for each client.

Master prompt

Reusable main prompt template for onboarding each new client
